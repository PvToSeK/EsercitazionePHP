Le istruzioni preparate PHP MySQL vengono utilizzate principalmente per prevenire attacchi di SQL injection e per migliorare le prestazioni.

Le istruzioni preparate separano i dati dai comandi SQL.

Le dichiarazioni preparate funzionano sostanzialmente in questo modo:

Preparazione: Un modello di query SQL con segnaposto viene inviato al server. I valori dei dati non vengono inviati. Esempio: INSERT INTO MyGuests VALUES(?, ?, ?). Successivamente, il server analizza, compila e ottimizza il modello di query SQL, senza eseguirlo.
Esecuzione: Successivamente, l'applicazione associa i valori ai parametri e il database esegue la query. L'applicazione può eseguire la query tutte le volte che vuole con valori diversi.
Le dichiarazioni preparate presentano quattro vantaggi principali:

Tempo di analisi ridotto : la preparazione della query viene eseguita una sola volta (anche se l'istruzione viene eseguita più volte).
Riduzione al minimo della larghezza di banda : i parametri associati riducono al minimo la larghezza di banda verso il server, poiché è necessario inviare solo i parametri ogni volta e non l'intera query.
Sicurezza - Le istruzioni preparate sono molto utili contro le iniezioni SQL, perché i valori dei parametri, che vengono trasmessi successivamente utilizzando un protocollo diverso, non necessitano di essere correttamente sottoposti a escape. Se il modello di istruzione originale non deriva da input esterni, non può verificarsi un'iniezione SQL.
Codice più pulito : separando i dati dai comandi SQL.
